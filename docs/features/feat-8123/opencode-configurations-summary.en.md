This note explains the durable design rules for OpenCode configuration, with `dot_config/opencode/opencode.json.tmpl` as the source of truth for the live setup. It defines stable agent naming and workflow handles, default model choices, and provider routing decisions for GPT, Gemini, Grok, GLM, and local models. It also draws a clear boundary: this document should preserve long-term conventions and intent, not temporary status, toggles, or provider health snapshots.
