## @pm expectations

When this PRD is assigned to a PM agent, the agent should:

1. **Review for completeness** - Ensure all sections are filled out appropriately
2. **Validate technical approach** - Confirm the technical solution is sound
3. **Check scope clarity** - Verify in-scope and out-of-scope are clearly defined
4. **Identify risks** - Surface any risks or concerns
5. **Estimate effort** - Provide rough effort estimates
6. **Suggest improvements** - Recommend any clarifications or improvements
7. **Approve or request changes** - Either approve or provide specific feedback

The PM should NOT:
- Rewrite the entire document
- Change the core feature concept without discussion
- Add scope without justification
- Remove critical requirements without discussion
