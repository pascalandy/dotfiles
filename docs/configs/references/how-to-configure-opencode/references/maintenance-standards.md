# OpenCode Maintenance Standards

Use this file for repository-level cleanup and documentation review.

## What to check

- Inventory of agents, commands, and skills
- Broken or missing frontmatter
- Duplicate skills with overlapping trigger surfaces
- Stale documentation after adding new OpenCode assets

## Preferred approach

- Fix metadata first.
- Preserve working behavior unless the user asked for a redesign.
- Treat audits as review tools, not permission to auto-rewrite everything.
