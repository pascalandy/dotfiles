# StandardPostmortem

Use for incidents that need a complete blameless review.

## Method

1. Summarize the event in one short paragraph.
2. State impact: users, systems, duration, and severity.
3. Build a factual timeline.
4. Identify root cause and separate contributing factors.
5. Record how the issue was detected and how the response unfolded.
6. Capture fixes already applied and follow-up actions still open.

## Output

Use the structure from `references/templates/IncidentPostmortemTemplate.md`.
