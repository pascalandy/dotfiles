# Memory Type Rubric

## observation

Use for preferences, patterns, habits, recurring concerns, and anything noticed that should influence future work.

## decision

Use for a chosen path, especially when the rationale matters later.

## learning

Use for a lesson that changes how future work should be approached.

## error

Use for a mistake, misread, unnecessary step, or avoidable failure.

## action

Use for a concrete step taken that is worth preserving as a record.

## thought

Use for internal reasoning worth keeping when it explains a future-relevant pattern.

## project_status

Use for the state of the project, artifact, or workstream at the end of the session.

## command_summary

Use for important tool-driven execution summaries that should be remembered later.
