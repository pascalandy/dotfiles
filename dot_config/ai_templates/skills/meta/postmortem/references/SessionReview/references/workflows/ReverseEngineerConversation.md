# ReverseEngineerConversation

Reconstruct the session before drawing conclusions.

## Inputs

- Conversation transcript, notes, or remembered sequence of work
- Any user instruction that defines the postmortem scope

## Method

1. Rebuild the sequence of events in plain language.
2. Identify the meaningful pivots: requests, decisions, course corrections, mistakes, discoveries, and fixes.
3. Drop details that are not likely to matter later.
4. Produce a compact `What We Did` section that is chronological and factual.

## Output

- A short chronological summary of the session
- Enough context for later memory extraction
