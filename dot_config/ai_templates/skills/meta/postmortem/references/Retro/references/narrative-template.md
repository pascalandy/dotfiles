# Retro Narrative Template

## Output Structure

```
Week of [date range]: [N] commits ([M] contributors), [X]k LOC, [Y]% tests, [Z] PRs, peak: [H]pm | Streak: [D]d

## Engineering Retro: [date range]

### Summary Table
| Metric | Value |
|--------|-------|
| Commits to default | N |
| Contributors | N |
| ... | ... |

### Trends vs Last Retro (if prior exists)
| Metric | Last | Now | Delta |
|--------|------|-----|-------|
| Test ratio | X% | Y% | +/-Zpp |
| ... | ... | ... | ... |

### Time and Session Patterns
[Narrative: peak hours, session patterns, active coding time estimate]

### Shipping Velocity
[Narrative: commit type mix, fix-chain detection, PR sizes]

### Code Quality Signals
[Narrative: test ratio trend, hotspot analysis]

### Focus and Highlights
[Narrative: focus score, ship of the week]

### Your Week (personal deep-dive)
- Personal stats
- Session patterns and peak hours
- Biggest ship
- What you did well (2-3 things)
- Where to level up (1-2 suggestions)

### Team Breakdown (if multiple contributors)
#### [Teammate Name]
- What they shipped (2-3 sentences)
- Praise (1-2 specific things)
- Opportunity for growth (1 constructive suggestion)

### Top 3 Wins
1. [Win 1: what, who, why it matters]
2. [Win 2]
3. [Win 3]

### 3 Things to Improve
1. [Specific, actionable improvement]
2. [Improvement]
3. [Improvement]

### 3 Habits for Next Week
1. [Small, practical habit <5 min to adopt]
2. [Habit]
3. [Habit]
```

## Writing Guidelines

- Lead with the point. Say what happened, why it matters.
- Anchor praise and growth areas in actual commits and data.
- Frame growth areas as investments, not criticisms.
- Be specific: name the file, the function, the exact number.
- Keep the tweetable summary to one line.
- The "Your Week" section gets the deepest treatment.
