# DesignTheApproach

Use this workflow when the direction is chosen and the remaining work is technical planning or plan hardening.

## Route Selection

| Need | Load |
|---|---|
| Turn a feature or requirements doc into an implementation plan | `../../ce-plan/SKILL.md` |
| Deepen an existing plan, strengthen sequencing, or add research-backed detail | `../../ce-plan/SKILL.md` |
| Review a requirements or plan document for coherence, feasibility, design, scope, or security gaps | `../../document-review/SKILL.md` |

## Expected Result

- A plan that names files, risks, dependencies, and test scenarios
- A reviewed document with auto-fixes and explicit judgment calls
- A clean handoff into execution without inventing behavior mid-flight
