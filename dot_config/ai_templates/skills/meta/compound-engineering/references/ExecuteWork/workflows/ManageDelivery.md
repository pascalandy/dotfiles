# ManageDelivery

Use this workflow when the implementation exists and the remaining work is branch, commit, PR, or documentation publication mechanics.

## Route Selection

| Need | Load |
|---|---|
| Create an isolated worktree or branch for parallel development | `../../git-worktree/MetaSkill.md` |
| Create a clean commit with a useful message | `../../git-commit/MetaSkill.md` |
| Commit, push, and open or update a pull request | `../../git-commit-push-pr/MetaSkill.md` |
| Prepare docs for GitHub Pages deployment | `../../deploy-docs/MetaSkill.md` |
| Clean stale local branches whose remote tracking branch is gone | `../../git-clean-gone-branches/MetaSkill.md` |
| Optimize coding agent permissions and allowlists | `../../claude-permissions-optimizer/MetaSkill.md` |

## Expected Result

- Changes moved into the right git state
- Delivery mechanics handled by the narrowest useful skill
- A branch ready for review, release prep, or sharing
