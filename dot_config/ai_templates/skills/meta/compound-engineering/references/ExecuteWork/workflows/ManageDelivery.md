# ManageDelivery

Use this workflow when the implementation exists and the remaining work is branch, commit, PR, or publication mechanics.

## Route Selection

| Need | Load |
|---|---|
| Create an isolated worktree or branch for parallel development | `../../../skills/git-worktree/SKILL.md` |
| Create a clean commit with a useful message | `../../../skills/git-commit/SKILL.md` |
| Commit, push, and open or update a pull request | `../../../skills/git-commit-push-pr/SKILL.md` |
| Prepare docs for GitHub Pages deployment | `../../../skills/deploy-docs/SKILL.md` |

## Expected Result

- Changes moved into the right git state
- Delivery mechanics handled by the narrowest useful skill
- A branch ready for review, release prep, or sharing
