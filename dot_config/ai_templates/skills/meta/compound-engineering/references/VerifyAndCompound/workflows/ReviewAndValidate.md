# ReviewAndValidate

Use this workflow when the work needs structured validation, review, reproduction, or targeted QA.

## Route Selection

| Need | Load |
|---|---|
| Review code changes before merge or before opening a PR | `../../../skills/ce-review/SKILL.md` |
| Reproduce and investigate a reported bug from an issue | `../../../skills/reproduce-bug/SKILL.md` |
| Run browser-focused validation on affected pages or flows | `../../../skills/test-browser/SKILL.md` |
| Build and validate an iOS flow in the simulator | `../../../skills/test-xcode/SKILL.md` |
| Tune Claude permission policy from observed workflow history | `../../../skills/claude-permissions-optimizer/SKILL.md` |

## Expected Result

- A reviewed or validated change set
- Concrete findings, repro details, or test evidence
- A clearer decision about what must be fixed before shipping
