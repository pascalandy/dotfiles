# ResolveFeedbackAndTrack

Use this workflow when review output already exists and the remaining work is to turn findings into concrete next actions, small follow-up fixes, or tracked backlog items.

## Route Selection

| Need | Load |
|---|---|
| Resolve GitHub PR comments and review threads | `../../resolve-pr-feedback/SKILL.md` |
| Hand substantial review-driven code changes back into implementation | `../../ExecuteWork/SKILL.md` |
| Create a durable todo entry for follow-up work | `../../todo-create/SKILL.md` |
| Triage an existing todo backlog | `../../todo-triage/SKILL.md` |
| Resolve pending todos in parallel | `../../todo-resolve/SKILL.md` |

## Expected Result

- Review feedback turned into concrete work, small follow-up fixes, or a clean handoff back to implementation
- A cleaner backlog with explicit ownership
- Review-driven work stays aligned to the right lifecycle owner
