# Operate And Improve

Use this workflow when the request is about keeping docs, process, memory, or the gstack environment itself current after or around delivery work.

| Request Pattern | Load This Skill | Why |
|---|---|---|
| update docs after shipping, sync README or ARCHITECTURE, clean drift | `../../document-release/SKILL.md` | Bring project docs back in sync |
| weekly retro, what did we ship, team reflection | `../../retro/SKILL.md` | Run the retrospective workflow |
| review or manage gstack learnings, memory, patterns | `../../learn/SKILL.md` | Maintain accumulated learnings |
| upgrade gstack, refresh the installation | `../../gstack-upgrade/SKILL.md` | Update the gstack environment itself |

Expected artifact: synchronized docs, a retro report, refreshed learnings, or an upgraded gstack install.
