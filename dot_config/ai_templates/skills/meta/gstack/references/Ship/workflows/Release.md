# Release

Choose the shipping skill that should release, deploy, or observe the rollout.

| Request Pattern | Load This Skill | Use When |
|---|---|---|
| ship, open PR, push, prepare release | `../../../ship/SKILL.md` | The user wants the standard shipping workflow |
| merge and deploy, verify production | `../../../land-and-deploy/SKILL.md` | The user wants merge-to-prod execution plus verification |
| watch rollout, canary check | `../../../canary/SKILL.md` | The user wants rollout observation after deployment |
| configure deploy flow before first use | `../../../setup-deploy/SKILL.md` | The user needs deploy workflow setup before shipping normally |
