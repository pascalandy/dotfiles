# Build And Inspect

Use this workflow when the request is about inspecting behavior, debugging, browser-driven investigation, or implementation from an already approved design direction.

| Request Pattern | Load This Skill | Why |
|---|---|---|
| bug, broken flow, why is this broken, investigate failure, root cause | `../../investigate/SKILL.md` | Run structured debugging before changing code |
| open the page, inspect the flow, click through, capture browser evidence | `../../browse/SKILL.md` | Use the browser specialist for live inspection |
| use my real Chrome, side panel, headed browser control | `../../connect-chrome/SKILL.md` | Shift browser work into the headed Chrome workflow |
| generate production HTML from an approved mockup | `../../design-html/SKILL.md` | Turn the chosen design into implementation-ready HTML |

Expected artifact: browser evidence, a diagnosis, or implementation-ready HTML tied to an approved design.
