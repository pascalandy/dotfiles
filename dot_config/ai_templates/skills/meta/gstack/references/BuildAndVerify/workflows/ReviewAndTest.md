# Review And Test

Use this workflow when the request is about diff review, QA execution, design polish with fixes, or getting a second opinion on code quality.

| Request Pattern | Load This Skill | Why |
|---|---|---|
| review this diff, pre-landing review, check the branch | `../../review/SKILL.md` | Run the primary production-minded code review flow |
| second opinion, codex review, cross-model review | `../../codex/SKILL.md` | Add independent model review pressure |
| QA the site, test and fix, validate the flow end to end | `../../qa/SKILL.md` | Run the fix-and-reverify QA loop |
| QA report only, test but do not fix | `../../qa-only/SKILL.md` | Produce a report without changing code |
| design audit with fixes, polish the live UI | `../../design-review/SKILL.md` | Review visual quality and fix what is found |

Expected artifact: findings, fixes, screenshots, health scores, or explicit approval to move toward release.
