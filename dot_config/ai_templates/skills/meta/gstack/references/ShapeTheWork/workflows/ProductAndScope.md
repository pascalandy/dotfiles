# Product And Scope

Use this workflow when the request is still about problem framing, ambition, wedge selection, or narrowing scope.

| Request Pattern | Load This Skill | Why |
|---|---|---|
| vague idea, what should we build, brainstorm, is this worth building, office hours | `../../office-hours/SKILL.md` | Start with problem framing and wedge selection |
| think bigger, founder review, ambition, challenge the scope, expand or reduce scope | `../../plan-ceo-review/SKILL.md` | Reframe the product and test strategic ambition |
| run all plan reviews automatically, review the plan end to end | `../../autoplan/SKILL.md` | Trigger the packaged planning review pipeline |

Expected artifact: a clearer product direction, tighter scope, or reviewed requirements document.
