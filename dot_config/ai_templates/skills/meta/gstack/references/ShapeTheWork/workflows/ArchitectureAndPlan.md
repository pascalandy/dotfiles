# Architecture And Plan

Use this workflow when the request is about architecture, sequencing, implementation approach, edge cases, or plan quality before coding begins.

| Request Pattern | Load This Skill | Why |
|---|---|---|
| architecture review, data flow, edge cases, test plan, implementation plan | `../../plan-eng-review/SKILL.md` | Lock the build approach before execution |
| review the plan from a design quality perspective, rate the UX in the plan | `../../plan-design-review/SKILL.md` | Stress-test the design quality encoded in the plan |

Expected artifact: an approved or revised implementation plan with concrete risks, diagrams, or test guidance.
