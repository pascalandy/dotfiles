# Framework Fundamentals — The Base Grammar

> Every other sub-skill in this collection builds on these primitives. If the user is new to game-theoretic analysis, or if the question is "what's actually going on here?" without a more specific frame, start here.

## Core concept

A **game** has three components:

1. **Players** — who is actually participating (often not who appears to be)
2. **Rules / constraints** — the boundary conditions; what is permitted, rewarded, punished
3. **Incentives** — what players actually get from winning (often not what they say they want)

> *Identify players, rules, and incentives, and you can predict how the game turns out.*

But games don't exist in a vacuum. They sit inside a **superstructure** — demographics, economics, culture, politics, religion, population size, wealth levels, external threats — that determines *which* game is actually being played. Change the superstructure, and the game changes, even if the players and stated rules look identical.

## The primitives this sub-skill teaches

| Primitive | Description |
|---|---|
| **Players / rules / incentives** | The atomic decomposition. Always ask these three questions first. |
| **Superstructure** | The macro-context that selects which game is live. Three classical types: early society, growing society, overpopulated society — each with its own native game. |
| **Nash equilibrium (and its failure)** | The "rational" stable state — and the observation that real humans rarely sit at Nash when *status* is in play. |
| **Status-as-hidden-incentive** | The most commonly mis-identified incentive. If behavior looks irrational, the real game is usually status, not the stated objective (sex, money, survival, etc.). |
| **Stated vs. actual motivations** | Every stakeholder has a stated reason and an actual reason. Reform that only addresses stated reasons always fails. |
| **Convergence point** | In any multi-stakeholder system (a school, a firm, a state), only changes inside the zone where stakeholders' *actual* incentives overlap can succeed. Outside that zone, reform is rejected. |
| **Multi-game play** | Every agent is playing several games simultaneously (status, money, safety, ideology). Tradeoffs between games explain most "irrational" behavior. |
| **Ostracism as primary constraint** | In human games, social ostracism is usually a stronger constraint than formal rules. Agents will break written rules to avoid ostracism. |
| **EOC triad (Energy / Openness / Cohesion)** | A society, firm, or team's health and trajectory can be read from three axes: energy (drive/ambition), openness (to outsiders and new ideas), cohesion (shared identity and trust). All three high = rise. All three decayed = fall. |

## Workflow: applying the lens

1. **Identify the players.** Not the declared players — the *actual* ones. Absent parties (regulators, creditors, future voters) are often decisive.
2. **Identify the rules.** Formal + informal. What gets rewarded? What gets punished? What's taboo?
3. **Identify the incentives.** For each player, separate *stated* from *actual*. Ask: what outcome would they accept as a win *and* defend to their peers? That's usually status.
4. **Identify the superstructure.** Demographics, wealth, external threat, dominant ideology. Which classical game does this superstructure typically select?
5. **Check for multi-game tradeoffs.** Which games are in tension for each player? Which game dominates under stress?
6. **Diagnose with EOC.** Energy / openness / cohesion — is the system rising or decaying?
7. **Find the convergence point.** Where do actual incentives overlap? That's the only place change is possible.
8. **Predict.** What does the equilibrium look like? What would move it?

## Output format

When applying this lens, produce:

- **Players:** bullet list, each labeled (stated role / actual role).
- **Rules:** formal + informal, in two sub-lists.
- **Incentives:** per player, stated vs. actual.
- **Superstructure:** one paragraph diagnosis.
- **Dominant game:** the single game that is actually being played, named plainly.
- **EOC reading:** one sentence per axis.
- **Convergence zone:** what kind of change could succeed.
- **Prediction:** the most likely equilibrium and what would move it.
