# Civilizational Lifecycle — Long-Arc Diagnostics

> Societies, empires, and dominant classes follow a recurring birth → maturation → collapse arc. This sub-skill gives you the tools to locate a given society on that arc and to read which signals matter.

## Core concept

Every civilization observed in the historical record has followed the same trajectory: **birth → maturation → collapse → (reset)**. The arc is not linear time — it is driven by:

- the **EOC triad** (energy, openness, cohesion) rising then decaying,
- **elite overproduction** — more aspirants to elite status than the system can absorb,
- **fertility dynamics** — wealthy, educated women's choice to have or not have children,
- **efficiency-to-resilience collapse** — systems optimized for the best case fail catastrophically in the worst case,
- and periodic **game resets** — revolutions, conquests, or demographic replacements that wipe the board and restart the superstructure.

## The primitives this sub-skill teaches

| Primitive | Description |
|---|---|
| **Three superstructure types** | Early society (cooperation/cohesion), growing society (arranged marriage / fertility maximization), overpopulated society (status game / fertility collapse). Each is a stage on the arc. |
| **Six-stage imperial lifecycle** | Cooperation → hereditary elite → factionalism → warring states → equilibrium → court politics. The late stages map to decadence and replacement. |
| **EOC rise/decay** | Energy, openness, cohesion all rise on the way up; all decay on the way down. The mechanism differs per axis but the co-movement is reliable. |
| **Elite overproduction** | Too many credentialed aspirants, too few elite slots. Excess aspirants become counter-elites. Almost every collapse has this signature. |
| **Fertility as collapse indicator** | The strongest historical early-warning sign: wealthy/educated women refusing to reproduce. |
| **Class-as-different-games** | Different classes play fundamentally different games — obey-authority vs. negotiate-with-authority — and success patterns differ accordingly. |
| **Game reset conditions** | Revolutions happen when (a) debt load is unbearable, (b) landlessness is widespread, (c) slavery or serfdom has concentrated power. These three together = reset. |
| **Efficiency vs. resilience** | Late-stage systems optimize for efficiency (best case). Resilient systems carry slack (worst case). Paradigm shift from efficiency to resilience is a late-arc imperative. |
| **Mercenary-replaces-empire pattern** | Marginalized outsiders invited in as mercenaries often end up conquering the empire that invited them (Goths, Mongols, many others). |

## Workflow: applying the lens

1. **Locate the society on the arc.** Which of the three superstructures / six stages fits best? Use demographics, wealth distribution, and marriage / fertility patterns as anchors.
2. **Read the EOC triad.** One sentence per axis. If all three are decayed, you are in a late-arc state.
3. **Check elite overproduction.** How many credentialed aspirants vs. elite slots? Where are the surplus aspirants going (counter-elite formation, ideological extremism, professional-grievance movements)?
4. **Check fertility.** Wealthy, educated women's fertility is the leading indicator. Below replacement = late-arc.
5. **Check game-reset conditions.** Debt, landlessness, slavery/serfdom. Any two of three = elevated risk.
6. **Diagnose efficiency vs. resilience.** Is the system optimized for calm waters or for storms? Late-arc systems are usually dangerously efficient.
7. **Project.** Given the arc position and the diagnostic reads, what is the likely trajectory over 10, 30, 50 years? What event class would accelerate or reset it?

## Output format

- **Arc position:** superstructure / stage.
- **EOC reading:** one sentence per axis.
- **Elite overproduction:** yes/partial/no, with evidence.
- **Fertility signal:** rate, trend, class pattern.
- **Game-reset conditions:** which of the three are present.
- **Efficiency vs. resilience:** paragraph.
- **Projection:** three horizons (10/30/50y) with main drivers.
