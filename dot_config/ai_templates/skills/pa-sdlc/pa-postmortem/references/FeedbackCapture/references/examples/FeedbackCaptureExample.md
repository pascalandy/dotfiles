# Feedback Capture Example

## Context

A small tooling task changed scope twice before the final shape was clear.

## Lessons

- Ask one short clarifying question when export requirements appear unstable.
- Keep the structure lean until the user's real workflow is explicit.

## What To Repeat

- Present the metaskill structure for approval before writing files.

## What To Avoid

- Designing persistence logic around temporary absolute paths.

## Feedback for the Agent

- Treat path corrections as scope corrections, not minor edits.

## Feedback for the User

- The dedicated write utility keeps the metaskill simpler and easier to reuse.
