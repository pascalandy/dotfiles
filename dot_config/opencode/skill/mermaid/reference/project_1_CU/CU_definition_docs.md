# Définition : Cas d'Utilisation et Action Système/Fonctionnalité

Un **cas d'utilisation** représente une interaction complète entre un acteur métier et le système pour atteindre une valeure d'affaires concrête.

Une **action Système/Fonctionnalité** est une opération spécifique réalisée dans le système. C'est un moyen technique pour arriver à une valeur d'affaires, mais n'a pas de valeur en soi.

Pourquoi cette distinction est importante? Pour les parties prenantes métier, il faut comprendre la valeur d'affaires plutôt que les détails techniques. Les cas d'utilisation démontrent le ROI et justifient les investissements, tandis que les actions système sont des moyens techniques pour y arriver.
