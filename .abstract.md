This repository is the source-of-truth `chezmoi` tree for personal dotfiles plus a substantial AI tooling surface. It combines classic shell and Git config, local executables, app config under `dot_config/`, agent runtime and template assets, and apply-time automation in `.chezmoiscripts/`. Use it when you need to locate the authoritative source behind something in `~/`, understand what `just` and `chezmoi apply` operate on, or distinguish active AI surfaces from archived reference material. Retrieval should start from operator files such as `AGENTS.md`, `README.md`, and `justfile`, then jump directly to the owning path rather than browsing broadly.

## Use this when
- You need the source path for a managed dotfile or config that ends up in `~/`.
- You need the main operator entry points for checks, apply flow, or repo conventions.
- You need active AI surfaces such as `dot_config/opencode/`, `dot_config/ai_templates/`, or `dot_pi/agent/`.
- You need to separate live automation from archived reference areas.

## Main areas
- `AGENTS.md`, `README.md`, `justfile` - repo entry points, workflow, and trust anchors.
- `dot_zshrc`, `dot_gitconfig`, `dot_Brewfile` - classic top-level dotfiles.
- `dot_config/` - application config, including OpenCode and shared AI templates.
- `dot_local/bin/` - user-maintained executables managed through chezmoi.
- `.chezmoiscripts/` - pre/post apply automation with side effects.
- `dot_pi/agent/` - active Pi agent configuration.
- `./.overview.md#reference-map` — Detailed map

## Out of scope
- `private_*` trees unless the task explicitly requires sensitive config.
- Archived/reference areas such as `bienvenue_chez_moi/`, `dot_config/ai_templates/commands_archives/`, and `dot_config/ai_templates/skills_archived/`.
- Generated, ignored, cache, or repo metadata state that is not an authoritative source file.
