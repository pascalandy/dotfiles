# Overview

## Corpus Type

`mixed`. The repo is centered on executable dotfile/config source, but it also contains agent-facing Markdown prompts and skills, reference docs, and backup-oriented automation. For exact answers, prefer the executable source over prose.

## Navigation Strategy

Start with `README.md` for repo-wide conventions, secret/template expectations, and the broad layout. When behavior is applied automatically, trust `.chezmoiscripts/run_before_sync.sh` and `.chezmoiscripts/run_after_backup.sh` over descriptive docs. For app-specific questions, go straight to the relevant file under `dot_config/` or a top-level dotfile such as `dot_zshrc`; use `docs/opencode/config_schema.json` as a lookup aid after opening the matching config.

## Retrieval Routes

- If the task is about shell behavior, read `dot_zshrc` first, then follow any referenced companion dotfiles only if the answer is incomplete.
- If the task is about sync/apply side effects, read `.chezmoiscripts/run_before_sync.sh` or `.chezmoiscripts/run_after_backup.sh` before inspecting the copied destination paths.
- If the task is about agent prompts, commands, or skills, read `dot_config/ai_templates/commands/` or `dot_config/ai_templates/skills/`, then use `.chezmoiscripts/run_after_backup.sh` to see where those files are published.
- If the task is about OpenCode behavior, read `dot_config/opencode/config.json` or the relevant plugin/tool file, then consult `docs/opencode/config_schema.json` for field semantics.
- If the task is about local helper commands or quality checks, read `justfile` for the entry command, then open the matching script in `dot_local/bin/`.

## Gaps and Cautions

- `private_*` trees exist but are intentionally not mapped here; inspect them only for tasks that explicitly involve private machine state.
- `.chezmoiignore` excludes repo metadata, caches, package manifests, and some support files from normal apply/diff flows; do not assume every tracked path is applied to `$HOME`.
- `dot_config/ai_templates/skills_archived/` is archival. Prefer active `skills/` unless you are tracing legacy behavior.
