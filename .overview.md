# Overview

## Corpus Type

`mixed`. Centered on executable dotfile/config source, but also contains agent-facing Markdown prompts and skills, reference docs, and backup-oriented automation. For exact answers, prefer executable source over prose.

## Navigation Strategy

Start with `AGENTS.md` for AI-agent conventions and repo structure. When behavior is applied automatically, trust `.chezmoiscripts/run_before_sync.sh` and `run_after_backup.sh` over descriptive docs. For app-specific questions, go straight to the relevant file under `dot_config/` or a top-level dotfile such as `dot_zshrc`.

## Retrieval Routes

- **Shell behavior**: Read `dot_zshrc` first, then companion dotfiles if incomplete.
- **Sync/apply side effects**: Read `.chezmoiscripts/run_before_sync.sh` or `run_after_backup.sh`.
- **Agent prompts/skills**: Read `dot_config/ai_templates/commands/` or `skills/`, then check `run_after_backup.sh` for publish targets.
- **OpenCode behavior**: Read `dot_config/opencode/` configs.
- **Local helpers/quality checks**: Read `justfile` for entry commands, then matching script in `dot_local/bin/`.

## Gaps and Cautions

- `private_*` trees are intentionally not mapped; inspect only for explicit tasks.
- `.chezmoiignore` excludes repo metadata, caches, package manifests from apply/diff.
- `dot_config/ai_templates/skills_archived/` and `commands_archives/` are archival. Prefer active directories.
- `bienvenue_chez_moi/` is archived reference material.

## Child Maps

No child maps needed. The repo is flat enough for direct navigation via the reference map above.
