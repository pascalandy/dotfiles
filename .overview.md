This is a mixed corpus with one dominant pattern: executable and applied configuration beats prose. The repository root is a `chezmoi` source tree that fans out into classic dotfiles, application config, local scripts, AI runtime assets, and automation scripts that add side effects during `chezmoi apply`. The main navigation problem is not coverage of every file; it is recognizing which path is authoritative for a given behavior and which areas are active versus archival.

## Reference Map

| Path | Role | Use when | Trust |
| --- | --- | --- | --- |
| `AGENTS.md` | conventions | You need repo rules, naming, active-vs-archived guidance, or edit constraints. | high |
| `README.md` | orientation | You need install, quick-start, operator workflow, or high-level repo map. | medium |
| `justfile` | operator entry | You need the main commands for checks, apply, audit, QA, or maintenance. | high |
| `.chezmoiscripts/run_before_sync.sh` | pre-apply automation | You need to know what gets synced back into source before apply. | high |
| `.chezmoiscripts/run_after_backup.sh` | post-apply automation | You need to know what gets copied/rendered/synced outward after apply. | high |
| `dot_zshrc` | shell authority | You need interactive shell behavior, aliases, PATH, and CLI setup. | high |
| `dot_config/opencode/` | runtime authority | You need active OpenCode config, plugins, prompts, or tooling. | high |
| `dot_config/ai_templates/` | shared AI source | You need shared commands, skills, or templates distributed to agent homes. | high |
| `dot_local/bin/` | executable helpers | You need custom scripts actually invoked from the shell. | high |

## Navigation Strategy

Start at the root only long enough to classify the request. If the question is about operating the repo, begin with `AGENTS.md`, `README.md`, and `justfile`. If the question is about behavior during sync or apply, move immediately to `.chezmoiscripts/` and treat the scripts as more authoritative than prose. If the question is about a specific tool or runtime, jump straight into the owning subtree under `dot_config/`, `dot_local/bin/`, or a top-level `dot_*` file. Use archive paths only when the task explicitly asks for historical or reference material.

Treat `dot_config/` as a directory hub, not a single thing to summarize. The active AI surfaces are narrow and explicit: `dot_config/opencode/`, `dot_config/ai_templates/commands/`, `dot_config/ai_templates/skills/`, and `dot_pi/agent/`. For classic workstation behavior, top-level dotfiles and `dot_local/bin/` are the fastest route. For anything that mentions applied paths in `~/`, convert mentally back to the chezmoi source path before reading.

## Retrieval Routes

- Shell, PATH, aliases, prompt, and login behavior: start with `dot_zshrc`, then adjacent top-level dotfiles such as `dot_zshenv.tmpl`, `dot_bashrc`, or `dot_bash_profile` only if needed.
- Git, package manager, or machine bootstrap questions: read `dot_gitconfig`, `dot_Brewfile`, and the corresponding `justfile` recipes.
- OpenCode or shared-agent behavior: read `dot_config/opencode/` or `dot_config/ai_templates/`, then confirm any sync/export behavior in `.chezmoiscripts/run_after_backup.sh`.
- Local command behavior: read `justfile` for the operator-facing entry point, then go to the owning script in `dot_local/bin/` or the referenced subtree.
- Apply-time surprises, drift, or sync behavior: read `.chezmoiscripts/run_before_sync.sh`, `.chezmoiscripts/run_after_backup.sh`, and `.chezmoiignore`.
- Historical comparisons or old patterns: check `bienvenue_chez_moi/`, `dot_config/ai_templates/commands_archives/`, or `dot_config/ai_templates/skills_archived/` only after exhausting active paths.

## Child Maps

- `dot_config/` - no child map; use direct routing to `opencode/` or `ai_templates/` because those are the only consistently high-value entry points.
- `dot_local/bin/` - no child map; script lookup is better handled by direct filename search.
- `dot_pi/agent/` - no child map; active surface is small enough for direct inspection.
- `.chezmoiscripts/` - no child map; there are only two authoritative run scripts.
- `bienvenue_chez_moi/` - no child map; archived reference only.
- `dot_config/ai_templates/commands_archives/` and `dot_config/ai_templates/skills_archived/` - no child maps; archived reference only.
